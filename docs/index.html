Getting started

From note to execution in four steps

Write a note. Claude reads it, builds a full execution plan, shows it to you, and waits for your go. Nothing runs until you approve it.

Step 01

Write your note

Each line starts with a capital letter prefix. G for goal, A for agent, T for task, W for checkpoint. Type it or photograph a handwritten note.

Step 02

Share it

Paste the note or upload your photo. Say "run my note" or "execute this" — NOTE recognises the format automatically.

Step 03

Review the plan

Claude surfaces the full execution plan — every agent, every tool, every capability it will use — before anything runs. No surprises.

Step 04

Say "go"

Agents execute in order. You pause at any W checkpoints you wrote. Each agent's output is saved to a brief file. Synthesis at the end.

G Map London opportunity landscape for brand launch
A Research Analyst
A Strategist
T Compile into a launch brief
F PDF report
ℹ️ No G? NOTE infers a goal from your agents and tasks, shows it to you, and asks you to confirm before running anything.
The system

19 letters across 5 layers

Each letter tells NOTE what kind of item it is and when it takes effect. Write them in any combination — order within the note determines W checkpoint positioning and B brief attachment.

🧭 Framing — G C D N

Read first. Applied globally to every agent in the run.

G
Goal

The north star. Everything else serves this outcome.

C
Context

Background every agent absorbs. Brand, history, product.

D
Directive

Non-negotiable constraints. "No food delivery mentions."

N
Note

Your annotation. Logged and surfaced in the plan — never executed.

⚡ Execution — O A T

Who does the work and what they produce.

O
Orchestrator

Team lead agent. Sets dispatch order, reviews every output, owns synthesis.

A
Agent

A purpose-built specialist — gets a full self-contained brief file before running.

T
Task

A concrete deliverable or synthesis step. Run after agents complete.

📦 Data — R K I B

Feed information into agents before they run.

R
Resource

A URL or file to fetch. Pulled via your browser before agents run.

K
Knowledge

A personal KB lookup. Retrieved before agents run. Flags if not found.

I
Input

Explicit data to pass — budget, targets, parameters.

B
Brief

Detailed spec. Attaches to the A item directly above it. Never forward.

🔀 Flow — P W L S

Control when and how things run.

P
Priority

Override run order. P1 runs before P2 before P3.

W
Wait

Human checkpoint. Positional — pauses exactly where it appears in the note.

L
Loop

Repeat an A or T N times or until a condition is met.

S
Schedule

Run at a future time or on a repeating cadence.

📤 Output — F E M V

Where results go and in what form.

F
Format

Output format: PDF, PPTX, DOCX, Markdown, Craft doc.

E
Export

Destination: Craft, Todoist, email draft, or folder.

M
Memory

Save a summary to your personal knowledge base after delivery.

V
Validate

Quality-check output against G and D before continuing.

Execution

Every run follows the same four phases

Nothing executes until you say go. Phase 2 — the plan — is always shown to you first.

Phase 00
👁

Input recognition

Photo or typed text — Claude reads it either way. Handwritten? Transcription shown for correction before anything runs.

Phase 01
📦

Parse & pre-load

R URLs are fetched via your browser. K items queried from your KB. All data lands before agents are briefed.

Phase 02
📋

Show the plan

Full execution plan surfaced: agents, order, every tool, every capability. You see it all. Adjust or say "go".

Phase 03
⚡

Execute

Brief files created. Agents run in order. W checkpoints pause for your review. Outputs saved to .md files.

Phase 04
✅

Synthesise & deliver

All outputs compiled into your F format. E routes to destination. M saves to KB. Final result in chat.

✅ Every agent gets a brief file. Before any agent runs, Claude creates a [agent-slug]-brief.md that is fully self-contained. The agent operates only from that file — no memory of prior sessions.
Agents

The Orchestrator (O)

When you add an O line, you're not picking a generic manager. You're creating a purpose-built team lead that's built specifically from your note and manages every specialist agent beneath it.

O — Orchestrator
e.g. "Lead Strategist"

📋

Dispatch planning

O is briefed first from the note context. Before dispatching, O builds each agent's brief itself, then produces a dispatch plan — ordered sequence, rationale, and acceptance criteria per agent.

🔍

Output review

After each agent, O reviews the output against its criteria. Approve → next agent runs. Flag → execution pauses. O never silently carries weak output forward.

🧩

Final synthesis

O reads all approved outputs and produces the final deliverable — coherent because O had full context from the start, not a mechanical stitch-job.

📚

O's brief is built first

O receives the raw note context — G, C, D, T, A list, W positions. O builds each agent's brief before dispatching them. The agent briefs are O's output, not its input.

Good to know

Tips & edge cases

Common questions from the first time through.

B always looks backwards

B attaches to the A item directly above it — never below. Write B on the line immediately after the A it's briefing. If B has no preceding A, it becomes a T.

W is positional, not terminal

W doesn't wait until "all agents are done". It pauses exactly where it sits in your note. Agents above run first; agents below wait for your approval.

K fallback

If your personal KB isn't connected or the item isn't found, you'll see ⚠️ K: [item] — not found in the plan. Provide it, or say "proceed without it". NOTE never invents KB content.

A always builds fresh

A items create purpose-built agents by default. To use an existing plugin, name it explicitly: A adops-suite Creative Director. Otherwise a new agent is always built.

No G? That's fine

NOTE infers a goal from your agents and tasks, labels it "Inferred goal — correct me if wrong", and asks you to confirm. Nothing runs until you say yes.

Training knowledge only

Want no external tools at all? Add D No web search — use existing knowledge only. Every agent runs on training knowledge and labels outputs accordingly.

Handwritten notes

Photograph your note. Claude reads it directly via multimodal. Messy handwriting, crossed-out lines, and shorthand are all fine. Claude shows its transcription before running anything.

Format capabilities

PPTX needs the pptx skill. PDF needs the pdf skill. DOCX needs the docx skill. These are declared as required capabilities in the Phase 2 plan before you approve.

O is optional

Without O, Claude manages dispatch in note order. Add O only when you want a domain-specific team lead reviewing the work at every stage, not just at the end.
